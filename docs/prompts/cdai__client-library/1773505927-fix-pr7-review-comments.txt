## Review Comment 2: PublicClient should use getContract + .read pattern
File: `packages/client/src/client.ts` line 67
> "i feel like you should use the getContract (from vanilla viem) and .read.<function-name> methods... it'd be consistent with the user client that way"

The PublicClient currently uses `publicClient.readContract()` directly. Instead, it should use `getContract()` from viem to create a contract instance, then use `.read.functionName()` — same pattern as the UserClient uses with getShieldedContract.

Reference pattern from poker app (`/home/c/code/seismic-workspace/apps/poker/packages/poker-viem/src/client/chain/lobby.ts`):
```ts
import { getContract } from 'viem'

const contract = getContract({
  address: contractAddress,
  abi: marchMadnessAbi,
  client: publicClient,
})

// Then use:
const count = await contract.read.getEntryCount()
const deadline = await contract.read.submissionDeadline()
```
