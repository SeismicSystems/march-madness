## What You Need to Build

### 1. Contract ABI (`src/abi.ts`)
Export the MarchMadness contract ABI as a const array. You can extract this from the contract or write it by hand matching the contract interface. Key functions:
- `submitBracket(sbytes8 bracket)` — payable
- `updateBracket(sbytes8 bracket)`
- `setTag(string tag)`
- `getBracket(address account) → bytes8` — view
- `submitResults(bytes8 results)`
- `scoreBracket(address account)`
- `collectWinnings()`
- `getEntryCount() → uint32` — view
- `getScore(address) → uint8` — view
- `getIsScored(address) → bool` — view
- `getTag(address) → string` — view
- `numEntries() → uint32` — view
- `submissionDeadline() → uint256` — view
- `results() → bytes8` — view
- `owner() → address` — view
- Events: BracketSubmitted(address indexed), TagSet(address indexed, string), BracketScored(address indexed, uint8), ResultsPosted(bytes8), WinningsCollected(address indexed, uint256)

IMPORTANT: For shielded types (sbytes8), use `bytes8` in the ABI — seismic-viem handles the shielding transparently.

### 2. Three Client Access Levels (`src/client.ts`)
Create three client classes/factories:

**PublicClient** — can only do transparent reads (`.treadContract()`):
- `getEntryCount()`
- `getResults()`
- `getSubmissionDeadline()`
- `getOwner()`
- After deadline: `getBracket(address)`, `getScore(address)`, `getIsScored(address)`, `getTag(address)`

**UserClient** — extends public + shielded writes and signed reads:
- `submitBracket(bracket: `0x${string}`)` — uses `walletClient.writeContract()` (shielded write), sends entry fee
- `updateBracket(bracket: `0x${string}`)`
- `setTag(tag: string)`
- `getMyBracket()` — before deadline: uses `walletClient.readContract()` (signed read); after deadline: transparent read
- `scoreBracket(address)` — score anyone's bracket after results posted

**OwnerClient** — extends user + owner functions:
- `submitResults(results: `0x${string}`)`

Each client should take a contract address and the appropriate viem/seismic-viem clients. Look at `/home/c/code/seismic-workspace/apps/poker/packages/poker-viem/` for reference on how seismic-viem is used.

## Important Notes
- Use `walletClient.writeContract()` for shielded writes (bracket submission), NOT `.twriteContract()`
- Use `walletClient.readContract()` for signed reads (reading own bracket before deadline)
- Use `publicClient.readContract()` (or `.treadContract()`) for transparent reads (after deadline, public data)
- The entry fee is 1 ETH (1000000000000000000 wei)
