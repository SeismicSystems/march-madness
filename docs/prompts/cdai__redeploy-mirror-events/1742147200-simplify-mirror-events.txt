## Context

The BracketMirror contract needs its events simplified to use slug (the stable identifier) instead of array index (an implementation detail). This is needed for a Redis-based indexer that will process these events.

## What to change in BracketMirror.sol

Look at contracts/src/BracketMirror.sol. Make these changes:

1. **Events** — change to use slug instead of index:
   - `EntryAdded(uint256 indexed mirrorId, string slug)` ← drop `entryIndex`
   - `EntryRemoved(uint256 indexed mirrorId, string slug)` ← replace `entryIndex` with `slug`
   - `BracketUpdated(uint256 indexed mirrorId, string slug)` ← replace `entryIndex` with `slug`

2. **In `removeEntry()`**: capture slug before swap-and-pop, emit it
3. **In `updateBracket()`**: emit slug instead of index
4. **In `addEntry()`**: emit slug instead of index (if it currently emits index)

## Create redeploy script

Create `scripts/redeploy-mirror.sh` that:
- Sources `.env` from repo root for `DEPLOYER_PRIVATE_KEY` and `VITE_RPC_URL`
- Uses sforge to deploy ONLY BracketMirror (not the other contracts)
- Updates the `bracketMirror` address in `data/deployments.json` (use jq or similar)
- The deploy script pattern can be found in `contracts/script/DeployAll.s.sol` and `scripts/deploy-testnet.sh`
- You may need to create a small forge script like `contracts/script/DeployMirror.s.sol` for deploying just the mirror
