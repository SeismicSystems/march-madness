### Step 2: Refactor useContract hook
The client library provides three client classes:
- `MarchMadnessPublicClient` — transparent reads (getEntryCount, getResults, etc.)
- `MarchMadnessUserClient` — shielded writes (submitBracket, updateBracket), signed reads (getMyBracket)
- `MarchMadnessOwnerClient` — owner functions (submitResults)

Refactor `useContract.ts` to use these classes instead of raw `walletClient.writeContract()` calls. The client library already handles the distinction between shielded writes, signed reads, and transparent reads.

However, note the web uses seismic-react hooks (useShieldedWallet) which give you the walletClient/publicClient. You'll need to construct the client library classes from these. Something like:
```ts
import { MarchMadnessPublicClient, MarchMadnessUserClient, MarchMadnessAbi, ENTRY_FEE } from '@march-madness/client';

// In the hook:
const publicMM = new MarchMadnessPublicClient(publicClient, CONTRACT_ADDRESS);
const userMM = new MarchMadnessUserClient(walletClient, CONTRACT_ADDRESS);
```

Check the client.ts source to see the exact constructor signatures.
