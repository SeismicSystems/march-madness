## What You Need to Build

### HTTP Server
A lightweight HTTP server that serves the indexed bracket data to the frontend React app.

### Endpoints

**GET /api/entries**
- Read the JSON index file and serve it
- Returns the full `EntryIndex` JSON (BTreeMap of address → EntryRecord)
- Set CORS headers for frontend access (Access-Control-Allow-Origin: *)

**GET /api/entries/:address**
- Return a single entry by address
- 404 if not found

**GET /api/stats**
- Return basic stats: total entries, number with brackets revealed, number scored
- Derive from the index data

**GET /health**
- Simple health check, returns 200 OK

### Architecture
```
src/
  main.rs     — CLI entry + server startup
  routes.rs   — Route handlers
  state.rs    — Shared state (file path, caching)
```

### Key Behaviors
- Read the index file on each request (or cache with a short TTL like 5 seconds)
- Handle the case where the index file doesn't exist yet (return empty object)
- File locking for reads (use shared/read locks via `fs2` to coexist with the indexer's write locks)
- CORS headers on all responses
- Configurable via CLI args: `--port`, `--index-file`

### CLI Interface (use clap)
```
march-madness-server --port 3001 --index-file data/entries.json
```
- Default port: 3001
- Default index file: `data/entries.json`

### Framework
Use `axum` — it's lightweight, async, and well-suited for this. With `tokio` as the runtime.
